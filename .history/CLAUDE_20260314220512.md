# Claude Development Rules

## First Step: Understand the Project

Before writing any code, you must fully understand the project.

1. Read the relevant parts of the **entire codebase**.
2. Review related **PDF documentation** if available.
3. Check the **UI folder** to understand the expected behavior and final result.
4. Search for existing implementations before adding new logic.

Do **not** create new code if the same logic already exists somewhere in the project.

---

## Implementation Workflow

When working on a task or issue:

1. Understand the problem described in the issue.
2. Search the codebase for relevant files and existing patterns.
3. Reuse or extend existing logic whenever possible.
4. Implement the necessary changes with minimal modifications.
5. Ensure the code follows all project rules.
6. Create a **clear and descriptive commit message** explaining what was changed and why.

---

## Go (Golang) Coding Standards

Follow modern **Go best practices** and stay consistent with the latest Go updates and features.

### Code Structure

* Write **simple, single-purpose functions**.
* Keep code readable and easy to maintain.
* Prefer clarity over cleverness.
* Avoid unnecessary abstractions.

### Performance

Take advantage of Go's strengths:

* Efficient runtime
* Concurrency (goroutines when appropriate)
* Simplicity
* Strong tooling

The project runs on **AWS Free Tier**, so code must be **resource efficient and lightweight**.

Always prefer solutions that:

* Reduce runtime overhead
* Use minimal memory
* Avoid unnecessary computation

---

## Error Handling

Use Go’s error handling properly.

* Always check returned errors.
* Do not ignore errors.
* Provide **clear and descriptive error messages**.

Error messages must include enough context to debug problems, such as:

* Request parameters
* Response body (when relevant)
* Status codes
* Operation being performed

The goal is that a developer can understand the failure **without additional debugging**.

---

## Code Style

* All **comments must be written in English**.
* Write clear and meaningful comments when necessary.

---

## General Development Principles

* Keep implementations **simple and focused**.
* Avoid rewriting existing working code unless it is optimized in .
* Prefer modifying existing logic instead of duplicating it.
* Maintain consistency with the existing project structure.
